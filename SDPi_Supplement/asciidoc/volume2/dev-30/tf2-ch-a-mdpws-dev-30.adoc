====== Message Semantics

`s12:Envelope/s12:Body/msg:GetMdibResponse/msg:Mdib/pm:MdDescription`:: The <<actor_somds_provider>>'s descriptive part of the MDIB.

`s12:Envelope/s12:Body/msg:GetMdibResponse/msg:Mdib/pm:MdState`:: The <<actor_somds_provider>>'s state part of the MDIB.

====== Context States

.R7001
[sdpi_requirement#r7001,sdpi_req_level=shall]
****
A <<actor_somds_provider>> shall enclose all context states in pm:GetMdibResponse messages.

.Notes
[%collapsible]
====
[NOTE]
--
This requirement restricts optionality in including context states in GetMdibResponse messages as specified in <<ieee_11073_10207_2017>>, C.57 GetMdibResponse:

[quote]
____
Since contexts might include privacy-related information, a SERVICE PROVIDER MAY decide to leave
the MDS contexts empty.
____

Since transmission of GetMdibResponse messages is required to be secured, there is no need for omitting context states to meet confidentiality.
